## Tags removed from a returned item

If a customer wants to return an item but the **tags are off**, ask whether they
**still have the tags**. If they do, the tags can be re-attached (strung back on)
and the return accepted. If the tags are gone but the item otherwise looks new, use
judgment and check with the boss for higher-value items. Items must also be clean,
unworn, and unwashed.

## Washed or worn items

A **washed item is not eligible** for return or exchange. A worn-but-unwashed item
is judged on condition; hygiene/undergarment items are never returnable. **Exception:
a manufacturer defect that only becomes visible after washing** (print peeling,
snaps breaking, faulty construction) is treated as a defect and **is covered** — ask
for a photo, contact the brand, and route via the defect flow. General shrinkage
from washing is **not** covered. When in doubt, ask the boss before promising
anything. See `warranty-and-defects.md`.

## Zipper "lock mode" before accepting a broken-zipper return

Before issuing a return label for a "broken zipper," check whether it is actually a
**zipper lock mode** (a user-operation feature on some zippers), not a defect. Offer
to send a short **video tutorial** showing how to release the lock (ask the team if
a saved clip exists for common zipper brands). If the zipper is genuinely defective
after the demo, proceed with the return via the defect flow.

## Partial refund for a missing accessory

If an item arrives **missing an accessory** that should have been included
(sunglasses, hat, belt, straps): first check inventory for a spare and ship it
separately if available (no return needed). If there's no spare, offer the customer
either a **partial refund** for the missing piece (the price difference between the
with- and without-accessory variants) or a **full return** of the item. Contact the
brand if it was a packaging error. Any refund amount is escalation-only.

## "Customer keeps the item" — agent discretion (low value only)

Agents have discretion to tell a customer to **keep** an incorrect or defective item
rather than return it, when the item is **low value**, the correct replacement has
already shipped, or the customer flagged an honest error. **Do not** gift high-value
items without checking the boss first. Framing it as "no need to send it back — it's
our gift to you" preserves goodwill while avoiding return shipping on cheap items.

## Combining returns from multiple orders in one package

A customer may ship returns from **different orders in the same package**. Tell them
to include a **note inside the package identifying each item and its order number**
so the warehouse can process each return correctly. Drop-off and mail options both
work (see `sale-season-and-pickup.md`).

## Color / photo discrepancy ("doesn't look like the photo")

Acknowledge the concern warmly first — **do not lead with "our images are taken
professionally,"** which reads as dismissive. If the color is **materially
different** from the listing (e.g., listed light denim, received dark denim) or the
listing had **wrong details** (sleeve length, features), that is a **store/supplier
error** → free return via the defect flow. If it is a subjective shade-perception
issue, a return is possible only within the normal window and at the customer's
expense (or warehouse exchange during a sale), and not at all if the item is final
sale.

## International / US-address customers

Some customers are based abroad (e.g., Israel) and ship to a US address (family
member or forwarding service). For returns, the customer ships to that US address
and that person mails it back, or they return in person at the warehouse while in
the US. There are **no prepaid international return labels**. See
`international-orders.md`.

## Escalations and the boss cell

For very upset customers, chargeback threats, manager requests, privacy issues
(received another customer's items), or order value over ~$200 with an unresolved
complaint: produce a **SENSITIVE** acknowledgment draft, flag it for immediate
human review, and do not make promises or binding commitments.
The boss cell **845-570-3569** is escalation-only and must be **confirmed with the
boss before sharing**. Chargebacks and disputes always receive a sensitive draft,
but are never sent or decided by the AI (see `refunds-and-disputes.md`).
