Buttons Bebe — Knowledge Base Architecture

Stage 4 production system: Git-managed Markdown → pgvector semantic search → LLM-drafted replies

KB Retrieval Path

  • • kb_client.py: thin stdlib HTTP client
  • • kb_service.py: localhost :8899, owns embeddings + DB
  • • pgvector: Postgres+HNSW, 169 chunks, 384-dim
  • • BAAI/bge-small-en-v1.5 (fastembed ONNX)
  • • ~53ms per semantic search

Live Webhook Flow

  • • Gorgias → Caddy TLS → server.py :8080
  • • pipeline.py: fetch ticket+messages+orders
  • • classifier.py: categorize + safety gate
  • • draft_engine.py: KB-grounded draft (dry-run)
  • • Telegram notifications to 2 recipients
  • • feedback.db: drafts, replies, comparisons

Overnight Batch Pipeline

  • • export: 12mo ticket export (47,918 msgs)
  • • clean: 44,672 cleaned, 3,246 quarantined
  • • pair: 13,271 customer→agent Q/A pairs
  • • dedupe: 12,238 clusters (7s, blocking+trigram)
  • • enrich: LLM canonical Q&A (in progress)
  • • write-back → kb/learned/ → Git

KB Content (Git source of truth)

  • • policies/: 8 docs (DRAFT pending Chaim)
  • • intents/: 22 confirmed customer intents
  • • faq/: 4 derived from real ticket export
  • • tickets/: 3 curated exemplar tickets
  • • learned/: auto-populated by write-back
  • • ingestion_worker: git diff → upsert

LLM Layer

  • • model_gateway.py: single choke-point
  • • Provider: Ollama Cloud (deepseek-v4-flash)
  • • Used by classifier + draft_engine + enrich
  • • Mock provider for offline testing
  • • ~733ms latency to Ollama Cloud

Infrastructure

  • • VPS: srv1766050 (Ubuntu, 1GB RAM)
  • • Caddy: HTTPS, reverse proxy, auth
  • • systemd: gorgias-webhook + gorgias-kb
  • • pgvector: Docker, localhost :5433
  • • backup.sh: timer-based, includes feedback.db
