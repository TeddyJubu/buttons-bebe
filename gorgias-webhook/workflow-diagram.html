Gorgias AI Support Agent — System Workflow

Buttons Bebe • Drafting, Feedback Loop & Knowledge Base • v1.0

Workflow A — Drafting

  • • Customer message triggers webhook
  • • 4 GET calls fetch ticket, conversation, customer, orders
  • • Classifier assigns Immediate / High / Low
  • • KB search: match → draft; no match → ask owner via Telegram
  • • LLM generates draft using KB + order data + conversation
  • • Draft posted as internal note (never to customer)
  • • Immediate priority: Telegram notification to owner

Workflow B — Feedback Loop

  • • Agent reply triggers webhook (from_agent=true)
  • • Reply saved to replies table (dedup by message_id)
  • • Linked to most recent unmatched draft
  • • Similarity computed (difflib SequenceMatcher)
  • • similarity >= 0.7: reply committed to kb/learned/ → KB
  • • similarity < 0.7: committed but tagged review_pending
  • • KB grows → better drafts on next customer message

Safety Gates

  • • Only write to Gorgias: internal notes (channel=internal-note)
  • • Never sends anything to the customer automatically
  • • All writes require HERMES_ALLOW_WRITE=1 + dry-run first
  • • Immediate priority: escalation only, no customer draft
  • • Webhook secret validated on every request
  • • Dedup on webhook retries (message_id check)

Telegram Notifications

  • • IMMEDIATE priority: ticket escalation with deep link
  • • KB gap (High/Low): "How should I respond?" + save answer
  • • Weekly review: summary report every Monday 9 AM
  • • Owner answers are saved to KB for future use

Workflow C — Weekly Review

  • • Cron job: every Monday at 9:00 AM
  • • Queries feedback.db for aggregate metrics
  • • Match rate, avg similarity, exact matches
  • • Flags low-similarity drafts and no-reply drafts
  • • Reports KB growth and gaps
  • • Owner reviews → updates KB / prompts / rules

Knowledge Base Sources

  • • Source of truth: Markdown in a Git repo
  • • 1. Policy / FAQ / ticket files (manual)
  • • 2. Agent replies (auto-committed to kb/learned/)
  • • 3. Owner Q&A (human-in-the-loop)
  • • Retrieval: Supermemory hybrid search (semantic + graph)
  • • Worker syncs git diff → Supermemory; Ollama Cloud drafts
