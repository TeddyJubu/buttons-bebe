## What package protection is

Package protection is an **optional $1.99 add-on at checkout** that covers a
package if it is **lost, stolen, or misdelivered** on the way to the customer. It
**sometimes auto-adds** to an order. It can be **removed and refunded** when it is
irrelevant — for example, on a local-pickup order (see
`order-changes-and-cancellations.md`, "Removing / refunding package protection").

## With package protection — covered

If the order **had package protection** and the package is lost, stolen, or
misdelivered, the customer is eligible for a **full refund**. The agent may confirm
that protection was purchased and that the issue is covered, but the refund itself
is **escalation-only** — do not promise or process the money in an automated draft;
route to a human per `refunds-and-disputes.md`.

## Without package protection — not liable

If the order **did not have package protection**, the store does **not take
responsibility** for a package that is lost or stolen after it leaves the
warehouse. Help the customer **file a claim with the carrier** (UPS / USPS / FedEx)
and have them check with neighbors, household, and building office. Do not promise a
refund — there is no coverage without protection, and any goodwill exception is a
human decision.

## Removing package protection on pickup orders

For a **local-pickup** order, package protection is unnecessary. If a customer asks
to remove it, remove/refund the $1.99 and reply after the action is done. See
`order-changes-and-cancellations.md` and
`../intents/intent-06-cancel-refund-package-protection.md`.
